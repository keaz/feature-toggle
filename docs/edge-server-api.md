# Edge Server API Documentation

The edge server now exposes autogenerated OpenAPI documentation and an interactive Swagger UI.

- Swagger UI: http://localhost:8081/docs
  - If running on a different host/port, replace accordingly.
- Raw OpenAPI JSON: http://localhost:8081/api-doc/openapi.json

## Endpoints

- GET /health
  - 200 OK when the edge server is connected to the backend stream
  - 503 Service Unavailable otherwise

- POST /evaluate
  - Request body (application/json):
    {
      "feature_key": "string",
      "environment_id": "string",
      "context": [{ "key": "string", "value": "string" }],
      "client_id": "string (optional)",
      "client_secret": "string (optional)"
    }
  - Response body (application/json):
    { "enabled": true | false }

- POST /ofrep/v1/evaluate/flags/{key}
  - OFREP single-flag evaluation.
  - Request body (application/json):
    {
      "context": {
        "targetingKey": "user-123",
        "plan": "premium"
      }
    }
  - Response body (application/json):
    {
      "key": "discount-banner",
      "value": true,
      "reason": "TARGETING_MATCH",
      "variant": "enabled"
    }

- POST /ofrep/v1/evaluate/flags
  - OFREP bulk evaluation for all cached flags known to the edge server.
  - Accepts `If-None-Match` for ETag cache validation and `flagConfigEtag` / `flagConfigLastModified` query metadata for OFREP event-triggered re-fetches.
  - Request body (application/json):
    {
      "context": {
        "targetingKey": "user-123",
        "plan": "premium"
      }
    }
  - Response body (application/json):
    {
      "flags": [
        {
          "key": "discount-banner",
          "value": true,
          "reason": "TARGETING_MATCH",
          "variant": "enabled"
        }
      ],
      "metadata": {
        "version": "etag-value"
      }
    }
  - Returns `304 Not Modified` with no body when `If-None-Match` matches the current flag-set ETag.

You can explore and try requests directly via the Swagger UI.
